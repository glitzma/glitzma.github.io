# postman使用

功能： 用于设置全局变量，设置项目变量，通过请求返回关键信息发送其它请求时携带

公司用的是body:我一般是在form-data中写请求参数的
                也可以在raw中直接放json 
                通过{{}}这种方式获取公开变量的值

var data = JSON.parse(responseBody)

postman.setGlobalVariable("access_token",data.result.access_token);

# postman 中的script

postman集成了一个强大的，基于nodejs的script引擎，借助它，您可以为requests和collections添加动态的行为。 这样就可以在编写test suite时，构建可以包含动态参数的request,在request之间传递数据等等。 您可以在流程中的两个事件中添加要执行的javasScript代码： 1.在发送request之前，编写pre-request script,定制化request. 2.收到response之后，用test script,处理返回的数据

//具体流程如下 

`pre-requestScript ===>>>  request ===>>> response ===> test script`

# Pre-request scripts

pre-request script就是一段在发送request之前执行的代码。

比如：随机URL参数，变化的request body 等。

这里要注意的是，pre-request script和test scripts 一样，都是javascript,同时，和angular js一样，可以用两个{{}}访问环境变量

# 递增的url参数

首先定义好全局变量

应用的时候：

localhost:9200/stock-record/_search?q=_id:{{xxId}}

表格形式设置为

_id:{{xxId}}

点击parameter下方的Pre-request scripts选项，编辑我们的脚本：

var temp = parseInt(postman.getGlobalVariable("xxId"));
temp += 1;
postman.setGlobalVariable("xxId", temp);

脚本的意思是先获取xxId,再给xxId自增1，所以我们点send,xxId会自动加1

[注] 所有的环境变量或者global变量，都是以字符串形式存储的，所有我们自增 操作需要先转化为int

# Tests script

通过 Tests script测试结果，保持我们之前的pre-request scripts 不变。

var temp = postman.getGlobalVariable("xhbxId");
tests["Body matches string"] = responseBody.has("\"_id\":\""+temp+"\"");

pm.test用于输出信息例:

pm.test(JSON.stringify(pm.response.to.be.ok))

上面输出的信息可以在Test Results中查看

Here are some examples:

// example using pm.response.to.have
pm.test("response is ok", function () {
    pm.response.to.have.status(200);
});

// example using pm.expect()
pm.test("environment to be production", function () { 
    pm.expect(pm.environment.get("env")).to.equal("production"); 
});

// example using response assertions
pm.test("response should be okay to process", function () { 
    pm.response.to.not.be.error; 
    pm.response.to.have.jsonBody(""); 
    pm.response.to.not.have.jsonBody("error"); 
});

// example using pm.response.to.be*
pm.test("response must be valid and have a body", function () {
     // assert that the status code is 200
     pm.response.to.be.ok; // info, success, redirection, clientError,  serverError, are other variants
     // assert that the response has a valid JSON body
     pm.response.to.be.withBody;
     pm.response.to.be.json; // this assertion also checks if a body  exists, so the above check is not needed
});
