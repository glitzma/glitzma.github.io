# 通过Webpack的code-spliting实现Vue组件异步按需懒加载

因为VueJs的核心是组件，通过Webpack打包工具最后会生成一个特别大的vendor.js文件，在用户网络不太好的情况下，想要在2s内加载完这个特大js文件几乎是不可能的，所以同样需要做按需懒加载，幸好，Webpack提供了code-spliting功能可以帮助我们做到这一点。

# 方法：

使用vue-cli脚手架构建一个vue项目，然后使用vue官网提供的方法，通过异步来引入组件，即可实现按需加载

Vue路由懒加载方法

实现方法：

//代码引用时
const Foo = () => import('./Foo.vue')

在路由配置中什么都不需要改变，只需要像往常一样使用 Foo：

const router = new VueRouter({
  routes: [
    { path: '/foo', component: Foo }
  ]
})

webpack 路由配置注意事项：

module.exports = {
   entry:  {
       app: ['babel-polyfill', './src/main.js']
   }, //如果写多个入品文件，会让网站同事加载多个文件
   output: {
        path: __dirname + '/src/build/',//打包后的文件存放的地方
        filename: "[chunkhash].build.js",//打包后输出文件的文件名
        publicPath: './build/'
    },
};
