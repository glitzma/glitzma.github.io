# arrow fn 与普通函数异同

ES6标准新增了一种新的函数：Arrow Function（箭头函数）。

# 1.外形不同

箭头函数使用箭头定义，普通函数中没有。

// 普通函数
function func(){
  // code
}
// 箭头函数
let func=()=>{
  // code
}

# 2. 箭头函数全都是匿名函数

普通函数可以有匿名函数，也可以有具名函数。

// 具名函数
function func(){
  // code
}
 
// 匿名函数
let func=function(){
  // code
}

代码说明如下：

  • （1）.第一个函数是具名函数，使用严格的函数声明语法创建。

  • （2）.第二个是匿名函数，其实它的本质是声明一个变量并赋值为一个匿名函数对象（函数没有名称）。

而箭头函数全都是匿名函数，代码如下：

// 箭头函数全都是匿名函数
let func=()=>{
  // code
}

# 3. 箭头函数不能用于构造函数

普通函数可以用于构造函数，以此创建对象实例。

function Glitz(webName,age){
   this.webName=webName;
   this.age=age;
}
let glitzZone=new Glitz("Glitz Zone",5);
console.log(glitzZone.webName);
// 输出： Glitz Zone

Glitz被用作构造函数，通过它可以创建对象实例。

# 4.箭头函数中this的指向不同

在普通函数中，this总是指向调用它的对象或者，如果用作构造函数，它指向创建的对象实例。

具体参阅如下两篇文章：

（1）.JavaScript this详解一章节 (opens new window)。

（2）.JavaScript构造函数一章节 (opens new window)。

# 4.1 箭头函数本身不创建this

也可以说箭头函数本身没有this，但是在它声明时可以捕获别人的this供自己使用。

特别说明：this一旦被捕获，以后将不再变化。

现在已经知道箭头函数中的this捕获而来，下面再来解决this从哪捕获而来。

箭头函数中的this是在它声明时捕获它所处作用域中的this。

下面通过代码实例演示一下。

var webName="Glitz Zone";
let func=()=>{
  console.log(this.webName);
}
func();
// 输出Glitz Zone

以上代码，箭头函数在全局作用域声明，所以它捕获全局作用域中的this，this指向window对象。

var webName = "Glitz Zone";
function wrap(){
  let func=() => {
    console.log(this.webName);
  }
  func();
}
wrap();

// 输出： Glitz Zone

代码分析如下：

  • （1）.wrap函数执行时，箭头函数被声明，wrap函数作用域中的this指向window对象。
  • （2）.于是箭头函数中的this指向window对象。
var webName = "Glitz Zone";
function Wrap(){
  this.webName="Google";
  let func=() => {
    console.log(this.webName);
  }
  func();
}
let owrap = new Wrap();
// 输出： Google

代码分析如下： -（1）.Wrap被用作构造函数。 -（2）.使用new调用Wrap函数之后，此函数作用域中的this指向创建的对象实例。 -（3）.箭头函数也随之被声明，此时捕获这个this。 -（4）.于是打印结果是"Google"，而不是Glitz Zone。

# 4.2 结合call、apply或bind方法使用

大家知道call或者apply能够改变普通函数中this的指向。

let web = {webName:"Glitz Zone"}; 
function fn() { 
  console.log(this); 
  console.log(this.webName); 
} 
fn.call(web);
// 输出: {webName:"Glitz Zone"}
// 输出： Glitz Zone

通过call将fn函数中的this指向对象web。

但是它对箭头函数不管用，代码实例如下：

var webName="Google";
let web = {webName:"Glitz Zone"}; 
let fn=()=> { 
  console.log(this); 
  console.log(this.webName); 
} 
fn.call(web);
// 输出： Window {postMessage:f, blur: f, focus: f,
// 输出:  Google

箭头函数中的this指向window对象，并没有被call所影响。

# 5. 箭头函数不具有arguments对象

每一个普通函数调用后都具有一个arguments对象，用来存储实际传递的参数。

但是箭头函数并没有此对象。

关于arguments对象可以参阅JavaScript arguments对象 (opens new window)。

# 6. 其它区别

  • （1）.箭头函数不能Generator函数。
  • （2）.箭头函数不具有prototype原型对象。
  • （3）.箭头函数不具有super。
  • （4）.箭头函数不具有new.target。

    return function(c){
        return b + c;
