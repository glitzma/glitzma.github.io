babel本身只关系语法，不关心api。 babel不会处理模块化的内容（webpack处理模块化）

babel与webpack区别

  • babel --> JS新语法的编译工具，不关心模块化
  • webpack --> 打包构建工具，是多个loader、plugin的集合，处理模块化

babel环境搭建

package.json添加内容

  "devDependencies": {
    "@babel/cli": "^7.7.5",
    "@babel/core": "^7.7.5",
    "@babel/plugin-transform-runtime": "^7.7.5", // 可以稍后安
    "@babel/preset-env": "^7.7.5"
  },
  "dependencies": {
    "@babel/polyfill": "^7.7.0", // 可以稍后安
    "@babel/runtime": "^7.7.5" // 可以稍后安
  }

babelrc文件配置

{
    "presets": [
        [
            "@babel/preset-env", 
            // 只转换新的javascript语法，不转换新的api，如：Iterator、Generator、Set、Map、Proxy、
            // Reflect、Symbol、Promise等全局对象，及一些全局对象上的方
            // 法（如：Object.assign）都不会转码
            // es6在Array上新增的Array.from方法，Babel也不会转码，如果想运行这个方法需要使用babel-Pollyfill来转换
            // polyfill的中文意思是垫片，所谓垫片就是垫平不同浏览器或者不同环境下的差异，让新的内置函数、实例方法等到低版本浏览器中也可以使用
            // 官方给出@babel/pollyfill和babel-runtime两种解决方案，来解决这种全局对象或方法不足的问题
            // babel-runtime适合在组件和类库项目中使用，而babel-polyfill适合在业务项目中使用
        ]
    ],
    "plugins": [
    ]
}

在terminal中执行下面命令，会将src/index.js中的es6编译成es5。在用webpack时用babel-loader将es6转成es5

npx babel src/index.js

preset-env是一堆plugin的集合，这样我们就不用一个个引入plugin了。如果不能满足我们的需求，我们可以在plugins处扩展引入

.babelrc基本配置

  • @babel-polyfill模块可以模拟完整的ES2015+环境
  • 这意味着可以使用Promise和WeakMap之类的新的内置对象、Array.from或Object.assign之类的静态方法，Array.includes之类的实例方法以及生成器函数
  • 它是通过向全局对象和内置对象的prototype上添加方法来实现的。比如运行环境中不支持Array.prototype.find方法,引入polyfill我们就可以使用es6的方法来编写了，但是缺点是会造成全局空间污染。
  • V7.4.0版本开始。@babel/polyfill已被废弃

babel-polyfill按需引入

{
    "presets": [
        [
          // 这样配置是corejs的按需引入
            "@babel/preset-env",
            {
                "useBuiltIns": "usage", // 按需引入。就不需要写在文件中手动全部引入了。
                "corejs": 3 // 现在版本是3
            }
        ]
    ],
    "plugins": [
        [
            "@babel/plugin-transform-runtime", // 解决全局变量污染问题babel-runtime
            {
                "absoluteRuntime": false,
                "corejs": 3,
                "helpers": true,
                "regenerator": true,
                "useESModules": false
            }
        ]
    ]
}

babel-polyfill

core-js并不支持generator语法，需使用时要单独引入。而babel-polyfill是两都的集合。

  • babel7.4之后弃用babel-polyfill 推荐直接使用core-js和regenerator

遇到问题：

  • 会污染全局环境
  • 如果做一个独立的web系统则无碍
  • 如果做一个第三方lib，则会有问题

解决方案：

  • 通过babel-runtime解决污染全局变量问题
  • 产出第三方lib要用babel-runtime

babel-runtime

需要安装babel-plugin-transform-runtime插件，启用插件后babel会使用 babel-runtime下的工具函数

module: {
    rules: [
        test: /\.js?$/,
        use: {
            loader: 'babel-loader',
            options: {
                presets: ["@babel/preset-env"],
                plugins: [
                    "@babel/plugin-transform-runtime",
                    {
                        corejs: 3 , // 如果用到了Promise会自动引入对应的polyfill 3
                        helpers: true, // 移除内联的babel helpers并自动引入babel-runtime/helpers
                        regenerator: true, // 是否开启gen函数转换成使用regenerator runtime来避免污染全局作用域
                    }
                ]
            }
        }
    ]
}
