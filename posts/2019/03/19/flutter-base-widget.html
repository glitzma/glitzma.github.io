# flutter组件

# widget概述-布局模型

# 布局 可以分两类

  • Single-child layout widgets： 父级widget中只能包含唯一一个子级widget 子元素叫child -> Widget
  • Multi-child layout widgets： 父级widget中能包含多个子级widget 子元素叫children -> List<Widget>

# Single-child layout widgets

  • Container: Container 可让您创建矩形视觉元素。container 可以装饰为一个BoxDecoration, 如 background、一个边框、或者一个阴影。 Container 也可以具有边距（margins）、填充(padding)和应用于其大小的约束(constraints)。另外， Container可以使用矩阵在三维空间中对其进行变换。
  • Center: widget 可以使其子widget在水平和垂直方向居中.
  • Padding: Container.padding 与 Padding 没有任何区别，Container把一些小组件组合到一个包中，方便使用。
  • Align: 将其子项与其自身对齐，并随子项调整大小
  • FittedBox: 根据合适的尺寸对子项进行缩放和定位
  • AspectRatio: 将子项调整为特定宽高比
  • ConstrainedBox: 对子项添加额外约束
  • Baseline: 根据子项的基线定位子项
  • FractionallySizedBox: 将子项调整为可用空间的一小部分
  • IntrinsicHeight: 将子项调整为子项内部高度（不建议使用）
  • IntrinsicWidth: 将子项调整为子项内部宽度（不建议使用，可以导致树的深度为O（N²）的布局推测传递）
  • LimitedBox: 当盒子不受限制时，限制其大小
  • Offstage: 组件放在树中，但不绘制不可用于命中测试，不占父元素空间。但会消耗电池和CPU时间
  • OverflowBox: 对其子项施加不同学其父级的约束，可能允许子项举出父级
  • SizedBox: 创建一个具有固定大小的框
  • SizedOverflowBox: 可指定固定大小，可能会溢出
  • Transform: 在绘制子项前支持转换
  • CustomSingleChildLayout: 可以确定子项的布局约束，并可以决定子项的位置。还可以确定父级的大小，但父级的大小不能依赖于子项的大小。

注： 大多数组件只是其它简单组件的组合，而不是继承而来的

# Multi-child layout widgets

  • Row、Column: 这些具有弹性空间的布局类Widget可让您在水平（Row）和垂直（Column）方向上创建灵活的布局。其设计是基于web开发中的Flexbox布局模型。可以通过mainAxisAlignment（主轴属性）和 crossAxisAlignment（垂直轴属性）来进行flex布局
  • Stack: 取代线性布局 (译者语：和Android中的LinearLayout相似)，Stack允许子 widget 堆叠， 你可以使用 Positioned 来定位他们相对于Stack的上下左右四条边的位置。Stacks是基于Web开发中的绝度定位（absolute positioning )布局模型设计的。
  • IndexedStack: 显示来自子项具有指定的索引，堆栈总是和最大的子项一样大
  • GridView: 可滚动的2D网格组件
  • Flow: 通过逻辑调整子项的大小和定位
  • Table: 表格布局
  • Wrap: 以水平或垂直方式显示其子项
  • ListBody: 沿着给定轴顺序排列子节点，一般用ListView
  • CustomMultiChildLayout: 利用委托调整多个子项的尺寸和位置
  • LayoutBuilder: 构建一个可以依赖父级大小的子件树 和Builder组件相似
  • ListView: 线性排列的可滚动列表组件
  • Expanded: 扩展Row，Column，Flex 。将可用空间根据弹性系数进行划分
