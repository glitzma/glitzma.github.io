# webpack文档

最基础的配置是 加一个 entry 一个 output，但是打包时候会出现黄色的错误提示。mode需要配置

mode 分：production 打包后会压缩代码， treeshaking 的支持 development 打包后不会被压缩

# loader 是什么？

css-loader、style-loadr、sass-loader、postcc-load、imports-loader、file-load、url-loader、babel-loader、@babel/preset-env、babel-polyfill等。

@babel/preset-env 将es6转成es5

babel-polyfill 转成低版本浏览器可以支持的语法

它主要处理模块打包时的问题，可以自定义loader处理一些业务上的公共问题，查阅文档地址

编写好自定义loader后也可以像第三方loader一样使用

TIP

遇到问题，文件中有引用嵌套时，加上了importLoaders:2

# plugin 是什么？

htmlWebpackPlugin、cleanWebpackPlugin、hotModuleReplacementPlugin、DllReferencePlugin、workbox-webpack-plugin

htmlWebpackPlugin打包结束后，自动生成一个html文件，并把打包生成的js自动引入到这个html中 可以指定template模板

cleanWebpackPlugin清除某个目录

HotModuleReplacementPlugin热更新文件

DllReferencePlugin插件去读取第三方依赖库。所以说它的打包速度会得到一个很大的提升。

workbox-webpack-plugin 进行pwa配置时使用

plugin是webpack在运行到某个生命周期hooks时要执行事情。一般是一个class，用的时候用new xxx

class copyRightWebpackPlugin {
  constructor () {
    console.log('插件被使用了')
  }
  
  apply (compiler) {
    // compiler 是webpack的一个实例
    compiler.hooks.emit.tapasync((名字, (compilation, cb) => {
       // 执行某些异步代码后需要调用cb回调
    }))
  }
}

写好后，也可以去webpack.config.js中的plugin中使用

# webpackDevServer可以提升开发效率

使用webpackDevServer 需要在webpack.config.js中做配置.


devServer:{
  contentBase: './dist',
  open: true,
  proxy: xx,
  port: xx,
  hot: true,
  hotOnly: true // 如：不刷新页面
}

如果只想刷新某个模块可以做单独处理

if(module.hot){
  module.hot.accept('./number', () => {
    document.body.removeChild(document.getElementById('xxxxx'));
    number();
  })
}

也可以通过node 编写一个服务通过 webpack-dev-middleware 不过需要做特别多的配置，为了方便的话，不建议使用

# treeShaking 是什么？

treeShaking把模块中没用的去掉，只支持ES module的方式，development mode是没有tree shaking功能的。

::: 某些引用不想被tree shaking掉，需要在package中进行配置 sideEffects:[@babel/polly-fill] 如果将参数设置为false则都会被tree shaking掉 :::

tree shaking运行进会把没有用到的先注释掉，当进行压缩时将这部分代码删除。

# webpack性能优化

  1. 跟上技术的迭代(Node, npm ,yarn) 要有最新的
  2. 尽可能少的用loader（loader范围要小）可以用 include 或者exclude指定
  3. 尽可能少的用plugin，如：dev环境不要压缩文件。 还要注意尽量使用官方推荐的性能比较好的，或者社区认可的插件。要保证插件的性能和可靠性
  4. resolve参数要配置合理
     "resolve": {
   extensions: ['js', 'jsx'],
   mainFiles: ['index', 'child'] // 指定默认查找路径
 }
  5. 使用Dllplugin指高打包速度（相同引用只打包一次）
  6. 控制文件大小 treeShaking splitChunksPlugin
  7. thread-load paraller-webpack happypack 多进程打包
  8. 合理使用sourceMap 做map的时候会让打包变得非常慢

    开发环境推荐： cheap-module-evel-source-map 生产环境推荐： cheap-module-source-map 进行线上调试

  9. 使用stats分析打包结果
  10. 开发环境用webpackDevServer提升速度，开发环境中把没用的插件全删除
